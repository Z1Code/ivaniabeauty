# Git and Delivery / Git y Entrega

## Commits
- Keep commits small, scoped, and traceable.  
  Commits pequenos, tematicos y trazables.
- Use clear messages: what changed, where, and why.  
  Mensajes claros: que cambio, donde y por que.
- Do not mix urgent fixes with broad refactors.  
  No mezclar fixes urgentes con refactors amplios.

## Controlled Risk / Riesgo Controlado
- If unexpected worktree changes appear, stop and validate before continuing.
- Do not use destructive commands (`reset --hard`, `checkout --`) without authorization.

## Task Closure / Cierre de Tarea
- Summarize modified files and functional result.
- State validations run and known pending limits.
- If critical validation is missing, call it out explicitly.
